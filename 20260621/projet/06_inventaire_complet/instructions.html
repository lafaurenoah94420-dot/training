Exercice 6 / 6 — Dictionnaires

Résumé d'inventaire

len, boucle, f-string

Le concept

Tu as maintenant plusieurs outils pour travailler avec des dictionnaires : lire une clé, écrire une valeur, tester la présence avec in, parcourir avec une boucle for. Il est temps de les combiner pour produire un résumé lisible.

Un inventaire de dictionnaire a deux informations distinctes : le nombre de types différents (combien de clés — avec len) et le nombre total d'objets (la somme de toutes les valeurs — avec une boucle). Une f-string permet d'assembler ces deux nombres dans un texte formaté.

inv = {"munitions": 30, "herbe": 2, "spray": 1}
types = len(inv)          # → 3 clés
total = 0
for cle in inv:
    total = total + inv[cle]   # → 33 objets
resume = f"Inventaire : {types} types, {total} objets"

Dans Resident Evil, le casier de la police affiche ce genre de résumé : combien de catégories de fournitures, et combien d'unités au total.

Instructions

Tu inventories le casier de la police dans Resident Evil. Chaque type de fourniture a un nom et une quantité. Tu veux afficher un résumé en une ligne.

La fonction resume_inventaire

Cette fonction reçoit un seul paramètre qui s'appelle inventaire. C'est le dictionnaire du casier. Chaque clé est le nom d'un type d'objet (un texte), chaque valeur est la quantité (un nombre).

La fonction doit d'abord calculer deux nombres. Le premier est le nombre de types différents dans l'inventaire — c'est le nombre de clés, obtenu avec len(inventaire). Le second est le nombre total d'objets — c'est la somme de toutes les valeurs, obtenue en parcourant le dictionnaire avec une boucle for comme à l'exercice précédent.

Ensuite, la fonction doit retourner un texte construit avec une f-string. Le format exact est important : le texte doit être "Inventaire : N types, M objets", où N est le nombre de types et M est le total d'objets. Respecte l'orthographe, les deux-points et les virgules exactement comme indiqué.

Pourquoi deux nombres différents ? Parce que len(inventaire) compte les clés (les types), pas les objets. Un casier avec trente munitions, deux herbes et un spray a trois types mais trente-trois objets au total. C'est la même distinction qu'à l'exercice un entre « types » et « quantité totale ».

Voici des exemples pour vérifier ta logique :

casier = {"munitions": 30, "herbe": 2, "spray": 1}
resume_inventaire(casier)
# → "Inventaire : 3 types, 33 objets"

resume_inventaire({})
# → "Inventaire : 0 types, 0 objets"

resume_inventaire({"spray": 5})
# → "Inventaire : 1 types, 5 objets"

Indices

💡 Indice 1 — Les deux calculs

Commence par compter les types avec len(inventaire). Ensuite, crée une variable total à zéro et parcours le dictionnaire pour additionner chaque valeur avec inventaire[cle]. Tu réutilises exactement la logique de l'exercice précédent pour la somme.

💡 Indice 2 — La f-string finale

Une fois que tu as les deux nombres, retourne le texte avec une f-string. Le modèle exact :

return f"Inventaire : {len(inventaire)} types, {total} objets"

Tu peux aussi stocker len(inventaire) dans une variable avant si c'est plus lisible pour toi.
