Exercice 5 / 6 — Fonctions

Deux fonctions

deux fonctions qui se complètent

Le concept

Dans un vrai jeu, une seule fonction ne suffit pas. Tu en écris plusieurs, chacune avec un rôle clair : une calcule les dégâts, une autre dit si le personnage est encore en vie.

function reste(a, b)
  return math.max(0, a - b)
end

function positif(n)
  return n > 0
end

Elles sont indépendantes, mais ensemble elles décrivent le combat.

Instructions

Ellie se fait attaquer. Le jeu doit recalculer sa vie après les dégâts, sans jamais descendre sous zéro, puis savoir si elle est encore vivante.

Écris tes fonctions avec function nom(...) — sans le mot local devant. Sinon le testeur ne les trouve pas.

La fonction degats

Cette fonction reçoit deux paramètres. Le premier s'appelle vie — les points de vie actuels. Le deuxième s'appelle montant — les points de dégâts reçus.

La fonction doit soustraire le montant à la vie, puis retourner le plus grand entre zéro et ce résultat. Comme ça, la vie ne devient jamais négative.

degats(50, 20)  -- → 30
degats(10, 40)  -- → 0   (10 - 40 = -30, ramené à 0)

La fonction est_vivant

Cette fonction reçoit un paramètre qui s'appelle vie. Elle doit retourner vrai si la vie est strictement supérieure à zéro, et faux sinon.

est_vivant(30)  -- → true
est_vivant(0)   -- → false

Quand tu lances lua tester.lua, tu dois voir :

✅ degats(50, 20) → 30
✅ degats(10, 40) → 0
✅ degats(0, 5) → 0
✅ est_vivant(30) → true
✅ est_vivant(0) → false
